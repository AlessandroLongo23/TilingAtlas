# 2k+1 for round {3,4,6}: catalogue check (2026-07-06, CC for AL)

**Question (AL).** Specialize the skeleton's round bound `s* ≤ 2k + 2c − 1` to the {3,4,6}
family, where the escape cost is `c = 1` (no 12-gon to escape), giving **2k+1**. Does it cover
every non-elongated (round) {3,4,6} tiling?

**Answer: on the catalogue, YES — 0 violations, tight at k=1,2. But it is evidence, not proof,
and it does not validate L6's traverse mechanism.**

## Method
- Data: `figures/data/galebach.json` (1352 tilings; 1248 have a measured `s*` in
  `thesis/figures/charts/weight-tightness.csv`). Coords are ℤ[ζ₁₂], basis `[1,ζ,ζ²,ζ³]`, ζ=e^{iπ/6}.
- Bravais from the Gauss-reduced `(T1,T2)`: **round** = hexagonal or square; **elongated** =
  rhombic/rectangular/oblique (the skeleton's Part-0 split — note rhombic is *elongated*).
- Polygon set from the angular gaps in each vertex's unit-edge star (60/90/120/150° = 3/4/6/12-gon).
  `{3,4,6}`-family ⟺ **no 150° gap** anywhere.
- Degenerate `t1002` (T1=T2=0) excluded.
- **Validation:** (i) all k=1 configs recovered correctly (honeycomb→{6}, 4.6.12→{4,6,12},
  3.4.6.4→{3,4,6}, 3.12.12→{3,12}, …); (ii) after fixing a float-boundary bug in the star search
  (unit neighbours landing one bucket past an integer edge), **0 incomplete stars**, and the round
  set partitions exactly: **313 = 228 {3,4,6} + 85 with-12-gon + 0 other**. Fixing the bug moved 8
  tilings out of a *false* "has-12" classification. Reproducers:
  `experiments/th10-round346-{check,verify3}.py`.

## The c-split is confirmed
- round **with a 12-gon** (c=2 ⟹ 2k+3): max s* `[5,6,7,9,10,11]`, 0 violations, tight at k=1
  (t1003 = 4.6.12, s*=5=2k+3).
- round **{3,4,6}** (c=1 ⟹ 2k+1): max s* `[3,5,6,7,9,9]`, tight at k=1 (t1006 = 3.4.6.4, s*=3)
  and k=2 (t2005).

So L5's escape-cost split `c ∈ {1,2}` shows up exactly as the `+2` additive split (2k+1 vs 2k+3),
and the two tight cases are the two "diameter" tilings (3.4.6.4 and 4.6.12). This part is a clean,
real confirmation.

## Why this is not a proof, and does not close L6
1. **Circularity.** Catalogue-consistency cannot bound the proven enumeration run — that assumes the
   completeness the run is meant to establish (cf. `th10-k3-feasibility-audit` §2).
2. **The envelope is loose for k≥3.** Round {3,4,6} max `[3,5,6,7,9,9]` vs 2k+1 `[3,5,7,9,11,13]` →
   slack `0,0,1,2,2,4`, growing. The growth is `~√k`, not linear. So "0 violations of 2k+1" is what
   you would see **whether or not** L6's `2(k−1)` traverse is the real mechanism — an O(√k) truth
   under a loose linear cover produces the identical zero-violation picture. The tight cases (k=1,2)
   are **traverse-free** (escape+jump+enter only); the `2(k−1)` term is never approached. This does
   **not** validate L6.
3. **Same crux.** To *prove* round {3,4,6} ⟹ 2k+1 you still need the period-quantization statement
   (shortest hexagonal period is 30°-grid-directional). Restricting to {3,4,6} only lowers `c` (2→1)
   and the det constant — it does not dissolve the wall.
4. **Feasibility.** The k=3 binding tiling is the round **12-gon** champion (s*=7, bound 2k+3=9),
   **not** a {3,4,6} tiling (max 6). So 2k+1 sharpens a sub-family but does not lower the k=3 pool
   budget; the 12-gon round case (2k+3) still governs.

**Net:** `2k+2c−1` is a valid, c-correct upper envelope (0 violations in both c-buckets), tight only
at k≤2. "True bound" overstates it — it is a loose linear cover over sublinear (√k) reality for k≥3.
The proof obligation and the k=3 lever are unchanged.
